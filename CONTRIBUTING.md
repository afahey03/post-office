# Contributing to Post Office

Thanks for investing your time in Post Office.

## Quick Setup

1. Fork and clone the repository.
2. Install dependencies:

```bash
npm install
```

3. Start the app:

```bash
npm run dev
```

4. Run checks before opening a pull request:

```bash
npm run lint
npm run typecheck
npm test
npm run build
```

## Development Workflow

1. Create a branch from main.
2. Keep each pull request focused on one change.
3. Add or update tests when behavior changes.
4. Update docs when user-facing behavior changes.

## Pull Request Checklist

- Clear title and concise description
- Link related issue when available
- Screenshots or short recording for UI changes
- All CI checks passing
- No unrelated refactors in the same PR

## What to Work On

Great contribution areas:

- UX polish in JSON and API tools
- Better error handling and edge cases
- Additional tests for utility functions and routes
- Documentation improvements

## Communication

Be respectful, assume positive intent, and focus feedback on code and behavior.
Please follow the project code of conduct in CODE_OF_CONDUCT.md.
